00009  ---------
00010 
00011  This is a "zero-copy" FIFO queue implimented on a circular store of pointers

00013  with a swap operation that exchanges the pointer to the input buffer with the
00014  pointer to the head/tail. On a push, the token buffer has been filled with
00015  data to enqueue.  On a pop, the token buffer is just a token; the contained
00016  data may be deleted or overwritten

00018  When pushing to a full queue, there are three behaviors defined.  The queue
00019  is optionally expanded (involves a block copy and a number of mallocs), or
00020  both the read and write point are advanced around the circular queue 
00021  overwriting the oldest enqueued data.  A push-try operation will fail
00022  when the queue is full, resulting in a no-op.  All push functions return
00023  1 when the queue is full and 0 othewise.
00024 
00025  Popping from an empty queue results in a no op.  The pop will return 1 to 
00026  indicate the queue was empty.
00027 
00028  The circular store is always sized as a power of two (non-zero).  Expand
00029  will cause the store to grow exponentially.
00030 
00031  Implimentation is reentrant but not threadsafe.  That is, the interface may be
00032  used by multiple threads, but synchronization is required for proper use of a
00033  given fifo instance.
