00008 //
00009 // Windows Vista and later has almost one-to-one compatibility with pthreads.
00010 // Here I provide a wrapper to give the functionality provided by both
00011 // libraries a common interface.  I try to follow the pthread symantics when
00012 // possible.  Below are some exceptions:
00013 //
00014 // Thread_Alloc() and Thread_Free()
00015 //
00016 //   - need alloc/free symantics because of how windows allocates 
00017 //     threads.
00018 //
00019 // Thread_Exit(unsigned)
00020 //
00021 //   - windows threads return a DWORD that is usually interpreted as an exit
00022 //     code.  Pthreads return a void* that may point to the result of some
00023 //     computation.  The pthread convention is maintained by Thread_Join().
00024 //     That is, threads return a (void*).
00025 //
00026 //     Thread_Exit causes a thread to immediately abort.  There's no real
00027 //     isomorphism between pthread and windows here.  The return type
00028 //     of a windows thread isn't wide enough to use as a pointer on 64-bit
00029 //     systems.  As a result, it doesn't get used for Thread_Join(). 
00030 //
